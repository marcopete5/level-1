Initializing a Git Repository with git init

What is git init?

  • Explanation: git init is the command used to initialize a new Git repository in your project directory. It sets up all the necessary files and folders that Git needs to start tracking changes in your project.
  • Example: Imagine you're setting up a new workspace for your project. git init is like flicking the switch to turn on version control for that workspace.
  • Why it matters: Initializing a Git repository is the first step towards using version control for your project. It allows you to track changes, collaborate with others, and manage your project more effectively.

Adding .gitignore to your project

In general, a rule of thumb is to just include files that you plan on manually updating/writing. There are always exceptions but we don’t need to worry about this too much for now.

While you won’t be using some of these files until later on in the course, it is extremely important to add this now so you don’t forget later on.

  • From the root of your git repository, create a file called .gitignore. you can either do this manually in your editor or use the touch command: 

cd /path/to/your/git/repo
touch .gitignore
  • Open it up in your IDE, add the following text, and save the file: 

node_modules
.env
.DS_Store

That’s it! So what do each of these mean?

  • node_modules is a folder where 3rd party code from the Node.js NPM library lives. We have a way of auto-generating them so there’s no need to save them in our project.
  • .env is a file that usually contains sensitive data such as API keys, server secrets. This won’t be relevant until much later, so don’t worry if you’re not sure what these are.
  • .DS_Store is just a MacOS-specific configuration file that we don’t need to track.
