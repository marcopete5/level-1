Git is a Distributed Version Control System

Lesson

Distributed:

  • Explanation: In a distributed system, files (like your code) are not just stored in one place. They're spread out across different computers.
  • Example: Your code can be on your own computer and also on a platform like GitHub.
  • Why it matters: This means that many people can work on the same project at the same time without stepping on each other's toes.

Version Control:

  • Explanation: Version control means keeping track of different versions of your project.
  • Example: Imagine you're writing a story. Every time you make a big change, you save a copy with a new name (like "story_v1", "story_v2", etc.).
  • Why it matters: If you mess something up, you can always go back to a previous version. It's like having a time machine for your code!

Benefits:

Distributed Systems:
  • Explanation: Because your code is not just on one computer, it's safer.
  • Example: If your computer crashes or gets lost, your code is still safe on GitHub.
  • Why it matters: You won't lose all your hard work even if something bad happens to your computer.
Version Control:
  • Merge Different Versions:
    • Explanation: Let's say two people make changes to the same project at the same time. Version control helps to combine those changes together smoothly.
    • Example: Imagine two people are editing the same document. Version control helps to make sure their changes don't overwrite each other.
    • Why it matters: It keeps everything organized and prevents chaos when multiple people are working on the same project.
  • Rollback to Previous Versions:
    • Explanation: If you make a mistake and your code stops working, version control allows you to go back to an earlier, working version.
    • Example: It's like hitting an "undo" button on your code.
    • Why it matters: You can experiment and try new things without worrying about messing up your project irreversibly.
