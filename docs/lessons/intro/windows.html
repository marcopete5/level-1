  1. In your terminal, run echo $PATH

Can‚Äôt access Ubuntu folder from Network

Copy & paste the following address into your file explorer‚Äôs address bar:

Incorrect Starting Directory in Ubuntu Terminal

If you see / instead of ~ before the prompt ($), open up the settings page in Windows Terminal, click on ‚ÄúOpen JSON file‚Äù and scroll down to the Ubuntu profile object.

Replace this:


"startingDirectory": "//wsl$/Ubuntu/home/<USERNAME>"

with the following:


"startingDirectory": "/home/<USERNAME>"

Permission Denied (when reading, writing, accessing folders in root directory)

Run this command in your root directory (~):


sudo chown -R USERNAME:USERNAME ~

Live Server Does Not Automatically Open Chrome

If you‚Äôre using WSL, open up your settings.json file and add this: 


"liveServer.settings.AdvanceCustomBrowserCmdLine": "/mnt/c/Program Files/Google/Chrome/Application/chrome.exe"

Create an SSH key

  1. Check to see if you already have an SSH key. Run cat ~/.ssh/id_rsa.pub.
    • If it prints out a long string of characters, skip to step 3.
  2. In terminal, run the following: ssh-keygen -t rsa -N "" -f id_rsa -C your_github_email@here.com (Making sure to replace your_github_email@here.com with the actual email you use on Github). If you're unsure which email you used to sign up for Github, check the Public email section of your Github profile settings
  3. Run cat ~/.ssh/id_rsa.pub and copy your new ssh key to the clipboard so you can add it to your GitHub account.

Set up your Git config

Enter the following (replacing the name and email with your actual name and GitHub email address): 


git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

This ensures you get credit on GitHub for your commits.
